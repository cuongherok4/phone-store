### Task 1.1: Cài đặt các packages cần thiết

Chạy lần lượt các lệnh sau trong thư mục dự án (`phone-store`):

```bash
# 1. Cài packages PHP
composer require spatie/laravel-permission
composer require intervention/image-laravel
composer require laravel/scout
composer require barryvdh/laravel-dompdf

# 2. Cài packages Frontend
npm install -D tailwindcss @tailwindcss/vite alpinejs
```

**Sau khi cài xong**, hãy chạy lệnh sau để publish một số file quan trọng (đặc biệt là Spatie Permission):

```bash
php artisan vendor:publish --provider="Spatie\Permission\PermissionServiceProvider"
```

### Task 1.2: Cấu hình file `.env`

Mở file `.env` và cập nhật nội dung như sau (thay `your_password` bằng mật khẩu MySQL của bạn):

```env
APP_NAME="Phone Store"
APP_URL=http://localhost:8000

DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=phone_store_v2
DB_USERNAME=root
DB_PASSWORD=your_password

MAIL_MAILER=smtp
MAIL_HOST=sandbox.smtp.mailtrap.io
MAIL_PORT=2525
MAIL_USERNAME=your_mailtrap_username
MAIL_PASSWORD=your_mailtrap_password
MAIL_ENCRYPTION=tls
MAIL_FROM_ADDRESS="no-reply@phonestore.com"
MAIL_FROM_NAME="${APP_NAME}"

FILESYSTEM_DISK=public

# Scout (search) - tạm thời dùng database driver để đơn giản
SCOUT_DRIVER=database
```

**Lưu ý**: Nếu bạn chưa có tài khoản Mailtrap, có thể để trống phần Mail trước và cấu hình sau.

---

### Task 1.3: Import database

Chạy các lệnh sau:

```bash
# 1. Tạo database
mysql -u root -p -e "CREATE DATABASE IF NOT EXISTS phone_store_v2 CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;"

# 2. Import dữ liệu từ file SQL
mysql -u root -p phone_store_v2 < phone_store_v2_updated.sql
```

Sau khi import xong, kiểm tra nhanh bằng lệnh:

```bash
php artisan db:show --database=mysql
```

hoặc mở phpMyAdmin để xem có đủ 24 bảng không.

### Task 1.4: Viết toàn bộ Migrations từ file SQL

Đây là phần quan trọng nhất trong 4 task này.

Vì bạn đã có sẵn file SQL rất chi tiết (`phone_store_v2_updated.sql`), cách tốt nhất và sạch nhất là **tạo các migration thủ công** thay vì dùng công cụ tự động (để kiểm soát tốt hơn và tuân thủ Laravel conventions).

**Kế hoạch thực hiện Task 1.4:**

4. `create_categories_table`
5. `create_products_table`
6. `create_attributes_table`
7. `create_attribute_values_table`
8. `create_product_variants_table`
9. `create_variant_attributes_table`
